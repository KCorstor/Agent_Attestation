One browser click → one real agent.mjs process

demo_dashboard.mjs orchestrates reset, the merchant server, and the mock lender script. agent.mjs alone performs HTTP 402 handling, builds the payment bundle, waits for terms, signs with Tempo, and completes the paid GET. The UI streams phases over SSE so you can narrate the story slide-by-slide.

Why two processes plus the agent?

server.mjs is the merchant (402 + paid resource). demo_dashboard.mjs is only the demo conductor: reset, start merchant, spawn agent, run mock release when the agent has emitted the bundle.

What is “real”?

HTTP 402, MPP challenge parsing, Tempo signing, and retry are real mppx behavior. “Lender” terms are simulated by writing JSON to disk — a stand-in for webhook / queue / signed URL in production.

Copy & steps in the UI

Phase titles and explanations are editable in demo-ui/steps.json. The live checklist mirrors agent + orchestrator events from the stream.
